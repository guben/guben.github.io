C标准库—标准I/O库函数

C标准库，对于文件的操作。

FILE类型，用来顶一个指针指向一个打开的文件。

FILE *fp=fopen(“file1”,”r”);

fopen/fclose

stdin/stdout/stderr

终端设备的读写也是I/O操作。UNIX一切接文件，在/dev下有特殊的设备文件和设备对应。

问：为什么我们可以直接使用printf和scanf，而不用打开文件，就可以对终端设备进行操作呢？

答：因为在程序启动时，在main函数还没开始执行之前，就会自动将终端设备打开三次，分别赋值给三个FILE*指针，stdin/stdout/stderr。这三个文件指针是在libc中定义的全局变量，在stdio.h中声明，printf向stdout写，而scanf从stdin读。

errno和perror函数

errno是在libc中定义的全局变量，表示错误码。很多系统调用函数在错误返回的时候将错误的原因记录在errno中。是一个整形变量，所有的错误码都是正整数。

而perror函数是用来将errno这个数值翻译成对应的错误字符的。

##fgetc/fputc以字节为单位的I/O

几点注意点：

  • 文件指针指向文件的一个位置，调用fgetc或者fputc之后，这个指针会后移一个字节的。
  • fgetc成功的时候，返回读到的一个字节，本来应该是unsigned char类型，但是规定是返回int类型，为啥呢？因为如果出错了或者读到文件末尾需要返回EOF(-1)保存在int中是0Xffffffff,如果读到字节是0xff返回为int就是0x000000ff，这样就区别开了。所以一定要用int来接收返回值，然后和EOF比较。

操作读写位置rewind/fseek/ftell

rewind(FILE *fp);会将fp移动到文件的开头。

fseek则是制定偏移

ftell返回当前的读写位置

以字符串为单位的IO函数fgets/fputs

以记录为单位的IO函数fread/fwrite

格式化IO函数printf/fprintf/sprintf/snprintf/scanf/fscanf/sscanf

C标准的I/O缓冲库

C标准库的I/O缓冲区有三种类型：全缓冲、行缓冲和无缓冲。当用户程序调用库函数做写操作时，不同类型的缓冲区具有不同的特性。

  • 全缓冲

    如果缓冲区写满了就写回内核。常规文件通常是全缓冲的。

  • 行缓冲

    如果用户程序写的数据中有换行符就把这一行写回内核，或者如果缓冲区写满了就写回内核。标准输入和标准输出对应终端设备时通常是行缓冲的。

  • 无缓冲

    用户程序每次调库函数做写操作都要通过系统调用写回内核。标准错误输出通常是无缓冲的，这样用户程序产生的错误信息可以尽快输出到设备。

fflush函数，把I/O缓冲区中的数据立刻传给内核，让内核写回设备。
